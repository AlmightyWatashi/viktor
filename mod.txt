1. Uploading/downloading a mod
------------------------------

When uploading a mod somewhere online, you can put all the game files from the config folder where they can all be reached via the same link (the program just adds "rooms.json" and so on to the end).

FOR EXAMPLE: if you use Github to store the files, all the files are accessible under "raw.githubusercontent.com/foo/bar/master/"

When downloading a mod, you can go to your setup.json (located in the config folder of wherever you installed the game, for example it might be "C:/Program files (x86)/Steam/steamapps/common/Viktor, a Steampunk Adventure/config"). In the third line of the setup.json file just type in the link to where you are downloading the mod from, FOR EXAMPLE: "update" : "raw.githubusercontent.com/foo/bar/master/",

Then in the game you can press F5 and the game automatically downloads all the new config files from the link to the local app data folder.

2. Rules of modding
-------------------

When the game loads, it searches for "setup.json", in the following folders:
1) in the root folder of the %localappdata% folder - where it might have been downloaded as an update/mod
2) in the root folder of the game executable - it's not here (unless you put it here)
3) in the config subfolder of the %localappdata% folder - it's not here (unless you put it here)
4) in the config subfolder of the game executable - where the non-updated game data is (to avoid cluttering the root folder)

setup - the first JSON map which is loaded in the game, consisting of the following variables:
	date - just to keep trach of things
	version - game version
		accepts: string or special word "debug" (which allows SPACE to toggle the debug mode)
	action - the metagame function that is started when the game loads
		accepts: one of the following strings:
			"menu X" - where X is the alias of the menu to load
			"new X Y" - where X is the alias of the game room to load and Y in the alias of the room entrance that starts the room
			"polite X Y Z" - the same as "new" but if there already is a game in progress, leads to menu Z
			"save X" - where X is the name of the savefile to save
			"load X" - where X is the name of the savefile to load
			"savemenu X" - where X is the type of the save menu to open - either "save" or "load"
			"exit" - exits the game
			"options" - toggles the options menu
	steam - whether the game uses steam achievements and steam cloud save
		accepts: boolean
	stupidjoke - whether the game exits with a crash report or not
		accepts: boolean
	fullscreen - whether the game starts in fullscreen (can be changed with F10 any time)
		accepts: boolean
	room_speed - the number of frames per second, the game is tailored to 30
		accepts: real
	image_speed - the default number of images per frame for an object, the game is tailored to 0.25
		accepts: real
	viktor_walk - the default image speed of player animations when walking as viktor, the game is tailored to 0.25
		accepts: real
	matilda_walk - the default image speed of player animations when walking as matilda, the game is tailored to 0.25
		accepts: real
	player_speed - the default speed of the player when moving in pixels per frame, the game is tailored to 7
		accepts: real
	constantSave - whether the game will autosave on every change or only on the [autosave] block option
		accepts: boolean
	update - the web address which hosts the files (for update purposes)
		accepts: url
	title - the window caption of the game when in windowed mode
		accepts: string
	cursor_wait - the cursor picture when an in-game dialogue is played out
		accepts: sprite asset name as string
	cursor_cutscene - the cursor picture when an in-game cutscene is played out
		accepts: sprite asset name as string
	coin - a map consisting of variables defining the in-game monetary system:
		default - the default response to using coins with an object
			accepts: string
		name - the inventory name for coins when there is only 1 coin in the inventory
			accepts: string
		description - the inventory description of coins when there is only 1 coin in the inventory
			accepts: string
		names - the inventory name for coins when there is more than 1 coin in the inventory
			accepts: string
		descriptions - the inventory description of coins when there is more than 1 coin in the inventory
			accepts: string
	clickToContinue - the description text during dialogue
		accepts: string
	chooseOption - the description text when the interface opens
		accepts: string
	changeColorScheme - the description text when the use hovers over different wallpapers
		accepts: string
	multiple - whether all other JSON lists (rooms, guis, menus) are packed into this one or in json maps in seperate files
		accepts: boolean
	letterbox - list of strings defining wallpaper RGB schemes in a "xxx,yyy,zzz" format (or "xxx,yyy,zzz,qqq,www,eee" format for dual-color schemes)
		accepts: string
	shader - default shader which the game graphics pass through when pressing f2 in debug mode
		accepts: shader asset name as string
	records - list of strings defining the description text when the user hovers over different music tracks
		accepts: string (with # for new line)

menus - a JSON list of menus describing buttons and backgrounds for them, with the following variables:
	alias - name by which that menu is called forth
		accepts: string or special words "savemenu" (for the save menu) or "options" (for the options menu)
	cursor - the mouse cursor in this menu
		accepts: sprite asset name as string
	background - background image for this 
		accepts: background asset name as string
	music - the music playing on this menu as a loop
		accepts: sound asset name as string or undefined
	rgb - background color
		accepts: string in "xxx,yyy,zzz" format
	buttons - list of maps defining menu buttons, each consisting of:
		action - the metagame function which gets called when the button is clicked
			accepts: one of the strings defined above
		picture - the picture of the button
			accepts: sprite asset name as string
		x,y - location of the button
			accepts: real
		text - text drawn on button
			accepts: string
		rgb - color of text drawn on button
			accepts: string in "xxx,yyy,zzz" format
	slots - a map of variables defining the save slots if the alias of the menu is "savemenu":
		frame - the picture frame of the save slots
			accepts: sprite asset name as string
		frameScale - the scale of the picture of the save slots
			accepts: real
		pictureScale - the scale of the game screenshot shown in the frame
			accepts: real
		xPictureOffset, yPictureOffset - the offset of the picture with regard to the frame
			accepts: real
		xStart, yStart - the location of the first save slot
			accepts: real
		xOffset, yOffset - distance between two save slots
			accepts: real
		xMax - maximum x value before slots go to the next row
			accepts: real
		yMax - maximum y value after which the save slots are no longer created
			accepts: real
	records - a map of variables defining the position of soundtrack records if the alias of the menu is "options":
		x,y - location of the first record
		xOffset, yOffset - distance between two records
	colors - a map of variables defining the position of wallpaper buttons if the alias of the menu is "options":
		x,y - location of the first wallpaper button
		xOffset, yOffset - distance between two wallpaper buttons

guis - a JSON list of guis describing various interfaces, with the following variables:
	alias - name by which that gui is called forth 
	order - what is the default order of actions on an object if the gui is streamlined (like in Broken Age)
		accepts: string (coded in "x,y,z,..." format) or undefined (the gui is not streamlined)
	showInterfaceOnClick - whether an interface pops up when clicking on an object (e.g. like in MI3) or actions are defined as buttons (e.g. like in MI1)
		accepts: boolean or undefined
	interfacePicture - background sprite for the interface which opens when the player clicks on an interactable object
		accepts: sprite asset name as string
	interfaceScale - image scale of the interface background sprite
		accepts: real
	interface - list of maps containing variables about the interface buttons:
		alias - an alias which corresponds to an action button
			accepts: string
		picture - a picture of the interface button which corresponds to this action
			accepts: sprite asset name as string
		x,y - offset of button with regard to the interface background image
			accepts: real
	foreground - what is the GUI foreground image
		accepts: background asset name (as string)
	background - what is the GUI background image
		accepts: background asset name (as string)
	inventory - a map with the following variables regarding the inventory:
		x,y - starting position of inventory items
			accepts: real
		xOffset, yOffset - distance between two inventory items
			accepts: real
		xMaximum - maximum x value before items go to the next row
			accepts: real
	description - a map with the following variables regarding the description box:
		x,y - starting position of description text
			accepts: real
		rgb - color of description text
			accepts: string in "x,y,z" format
	default - default left click action on object
		accepts: button alias (as string)
	secondary - default right click action on object
		accepts: button alias (as string)
	actions - a list of buttons that start player actions, each consisting of:
		alias - name by which that action button is called forth
			accepts: string or special word "look" (which adds "[mood][cloud think][animation player spr_{PLAYERALIAS}]" to the front of game blocks)
		x,y - location of the button on screen
			accepts: real
		type - type of button
			accepts: one of the following strings: 
				"movement" - when a position is selected on the screen, move to its position without starting any interaction
				"observation" - when an object is selected on the screen, start the designated interaction without moving toward it (such as the LOOK action)
				"interaction" - when an object is selected on the screen, move to that objects interaction location and start the designated interaction 
				(or don't move if moveForInteraction is false in rooms.json, then it's just like observation)
				"combination" - pick 1) an inventory item and 2) an object; move to that objects interaction location and start the designated interaction 
				(or don't move if moveForInteraction is false in rooms.json)
				"immediate" - immediately start the interaction defined in this actions default variable
				scr_setRoom line 225
		default - what is the default player reaction with an object for this action
			accepts: string (coded as game blocks)
		available - is this action available (can be used to temporarily deactivate an action)
			accepts: boolean
		visible - is the action button visible on the gui
			accepts: boolean
		description - text which appears before the object/item name as a description when mouse hovers on an object with this action activated
			accepts: string
		combinationDescription - text which appears after the item name as a description when mouse hovers on an object with this action activated
			accepts: string or undefined
		name - text which appears as a description of the action when mouse hovers on the action button
			accepts: string
		binding - which keyboard keys correspond to which button actions
			accepts: string of characters (that can be pressed on a keyboard) in a "xyz..." format
		picture - image of the action button on the gui
			accepts: sprite asset name as string
		passive - image of the cursor when this action is activated and no object is under the cursor
			accepts: sprite asset name as string
		active - image of the cursor when this action is activated and an interactable object is under the cursor
			accepts: sprite asset name as string

rooms - a list of rooms describing game levels (rooms) with the following variables:
	alias - name by which that room is called forth
		accepts: string
	texture - which texture group does this room belong to?
		accepts: undefined, any string or special word "previous"
	border - transparent sprite used to determine border of walking in the room
		accepts: sprite asset name as string
	xBorder, yBorder - offset of the border sprite
		accepts: real or undefined
	background - background image for the room
		accepts: background asset name (as string)
	xBackground, yBackground - offset of the background image
		accepts: real or undefined
	backgroundScale - scale of the background image
		accepts: real or undefined
	gui - which user interface to load for this room
		accepts: gui alias as defined in guis.json (as string)
	rgb - background color for the room
		accepts: string in "x,y,z" format or undefined
	moveForInteraction - whether the player should move to object coordinates when interacting with the object
		accepts: boolean
	entrances - list of maps with variables defining how a room can begin, each consisting of:
		alias - name by which that entrance is called forth
			accepts: string
		entrance - game blocks that run at the start of the room
			accepts: string (coded as game blocks)
	objects - list of maps with variables defining all the objects in the room, each consisting of:
		alias - name by which that object is called forth
			accepts: string or undefined (if undefined, the alias becomes "token_alias")
		available - if the object is available when the room starts
			accepts: boolean
		visible - if the object is visible when available
			accepts: boolean
		idle - if the object starts a {PICTURE}_idle animation at random intervals and also (but more rarely) {PICTURE}_idleX (where X is a number) animations
		path - a predifend path for the object
			accepts: path asset name as string or undefined (if undefined, the object does not move)
		pathAction - what happens after the path is over
			accepts: strings "restart", "continue", "reverse" or "stop" or undefined (if undefined, it defaults to "restart")
		pathDelay - how long to wait before starting the path
			accepts: real or undefined (defaults to 0)
		depth - depth of the object layer, a higher value is further away from the screen
			accepts: real or undefined (if undefined, the depth changes according to the -y position)
		text - text drawn on button
			accepts: string or undefined (defaults to no text)
		rgb - color of text drawn on button
			accepts: string in "xxx,yyy,zzz" format or undefined (defaults to black)
		picture - the picture of the object
			accepts: sprite asset name as string or undefined (if undefined, the picture defaults to spr_{ALIAS} - if it exists)
		subimage - show a fixed subimage of the object instead of running the animation
			accepts: real of undefined (defaults to -1 meaning no fixed subimage)
		sounds - list of sounds for that object, each consisting of a string, the "default" variable being played in a loop while the object is available
			accepts: sound asset name as string or undefined (defaults to no sound)
		variables - list of additional variables for that object
			accepts: anything
		x,y - location of the object on screen
			accepts: real or undefined (defaults to 0,0)
		moves - if the object will move inside the room, if set to yes, the new position will be remembered instead of the old one when re-entering the room
			accepts: boolean or undefined (defaults to false)
		xInteract, yInteract - location where the player needs to stand to interact with the object
			accepts: real or undefined (defaults to x,y)
		changeInteractPosition - if the interact position is relative to the objects current x,y (useful when objects are moving)
			accepts: boolean or undefined (defaults to false)
		interactWithoutMoving - if the player should interact with this object without moving to its interact position
			accepts: boolean or undefined (defaults to false)
		moveInteract - same thing ???
		solid - if the player should avoid this object when moving around the room (the object receives a small collision mask)
			accepts: boolean or undefined (defaults to false)
		transparency - the alpha value of the object
			accepts: real or undefined (defaults to 0)
		angle - the angle (rotation) of the object
			accepts: real or undefined (defaults to 0)
		xScale, yScale - the scale of the object
			accepts: real or undefined (defaults to 1)
		banter - a list of strings that the object speaks at random intervals
			accepts: string or undefined (defaults to no idle banter)
		direction - direction the player should be facing when interacting with this object
			accepts: string "left" or "right" or "front" or "back" or undefined (defaults to automatic calculation of players direction)
		movementSpeed - default speed of motion for this object (when moving on a path or otherwise)
			accepts: real or undefined (defaults to 5)
		talkSoundOverride - default sound for speeches (a random sound asset is chosen which begins with the given string)
			accepts: sound asset name as string (defaults to snd_{ALIAS}_talk
		cloud - direction of the speech cloud for this object
			accepts: "left" or "right" or undefined (defaults to "left" if the object is in the right part of the screen and vice versa)
		xCloud, yCloud - location of the speech cloud relative to the objects location
			accepts: real or undefined (defaults to 0,0)
		description - description text when the mouse hovers over the object
			accepts: string or undefined (defaults to "")
		order - defines the order of actions to be activated each time the object is interacted with (if the streamlined gui is selected in guis.json)
			accepts: string (coded in "x,y,z,..." format) or undefined (the order variable is taken from guis.json)
		inventory - a map defining an item, consisting of variables:
			alias - name by which the item is called forth
				accepts: string
			name - the in-game name of the item
				accepts: string
			picture - the picture of the item in the inventory
				accepts: sprite asset name as string
			description - description text when the mouse hovers over the item in the inventory
				accepts: string
			default - default game block that runs when the item is used with an object
				accepts: string (coded as game blocks)
			special - whether using the item prompts a special game block to run (such as the map or the phone)
				accepts: string (coded as game blocks) or undefined (no special game block runs when the item is used)
			combinations - a map consisting of variables where the variable names correspond to item aliases and action aliases
				accepts: string (coded as game blocks)

Game Blocks
	- strings
	- each game block in a string is seperated by a > sign
	- each game block can have options and / or speech
	- options are given at the beginning of a block and enclosed in [brackets]
	- speech is given after the options
- examples:
"this is a game block > this is another block > a game block contains some dialogues > it will be said out loud by the player"
"[some option] speech with one block option > [an option][another option] speech with two block options > speech without options"

List of block options
	The asterisk * means that the argument is optional and can be ommited.
	The uppercase letters are variables (so for example you wouldn't write [room X Y], you write [room garden wake] and so on).
	When referencing an object, use its alias, or use "player" for the player object, or use "other" for the object the player is interacting with.
	Here is a list and explanation of all the block options in the game and their arguments:
	
	[autosave] - autosave the game at this point (and make a screenshot, ideal for between levels
	[menu X] - end the current game and open a menu X
	[gamemenu X] - open a menu X without ending the current game
	[banter] - raise a flag that the current objects "idle banter" contains no text (otherwise an empty cloud appears)
	[clean] - destroy all objects which are not the parser and the player
	[minigame X] - destroy a minigame object with the alias X
	[minigame] - destroy all minigame objects
	[digout X Y] - like [room], but for use when exiting the tunnel minigame
	[safe randomize] - select an unopened safe which is not safeCurrent and set it to safeCorrect
	[safe select] - set safeCurrent to the alias of the currently chosen safe
	[safe open] - destroy the object of the currently locked safe and create an object of the opened safe
	[safe detect] - check if the safeCurrent is safeCorrect and if so, set safeUnlocked to 1 and show gold and glow objects
	[panel X] - for all obj_panel_screw set bool_screws = X and start their alarm 1
	[speed X] - set FPS to X
	[angle X Y] - for instance alias X set image angle Y
	[movement X] - whether the player moves when interacting or not
	[picture X] - fix the player object to one picture and don't change it
	[picture] - deactivate fixed picture for object
	[shader X] - create shader X in game
	[shader X Y] - create shader X in game with shader speed Y
	[shader] - destroy the current shader
	[occupied X] - set player objects occupied variable to X
	[switch X Y] - switch the player object with another object
	[change X Y] - change alias of object X to Y
	[alias X] - change alias of player to X
	[shake X Y] - shake the screen with magnitude X for duration Y
	[restart] - restart the game
	[finish] - end the game
	[finish X] - end the game and open a website X
	[remember] - remember the player objects exact location
	[recall] - put the player object on the remembered location
	[video X] - play a video (X accepts only intro.avi and logo.avi)
	[description X Y] - set description for object X to Y
	[zoom X Y] - zoom the camera by X with a speed of Y
	[fade X Y Z* W*] - create a fade object fading in or out (set X as "in" or "out") with speed of Y and depth of Z and with picture W
	[fade] - destroy the fade object
	[endfade] - the created fade object will cover the gui as well
	[path X Y Z* W* Q*] - after a delay of Q, set object X to path Y with speed Z and when the path ends do action W ("continue", "stop", "reverse", "restart")
	[cloud] - set positon of the speech cloud to the object you are interacting with
	[cloud X] - set the shape of the cloud to X where X is "talk", "think", "phone" or "reverse"
	[cloud X] - set the position of the cloud to the object X
	[cloud X Y] - set the cloud to position X,Y
	[cloud player X Y] - set the player objects default cloud position to X,Y
	[cloud dig X Y] - set positon of the speech cloud to X,Y relative to the mole in the tunnel minigame
	[cloud dig X Y Z] - set positon of the speech cloud to Y,Z relative to the digging object with alias X in the tunnel minigame
	[blocksound] - play no sound during the speech in this game block
	[blocksound X] - play sound X during the speech in this game block
	[sound X] - play sound X
	[sound X Y] - play sound X with delay Y
	[mute] - stop all sounds
	[mute X] - stop sound X
	[music] - stop all looping music
	[music X] - play X in a looping mode
	[soundtrack X] - stop current soundtrack and play the soundtrack snd_X
	[give] - remove active item from inventory
	[give X] - remove item X from inventory
	[record X] - add a record X to your gramophone collection (where X is just the composer name)
	[record X silent] - same as record but without the record visually going to the gramophone
	[active X] - set object X as active object
	[position X Y Z] - set object X to position Y,Z
	[position X Y] - set object X to the position of object Y
	[position X] - set object X to previous position
	[position] - set player to previous position
	[moveto] - stop all objects from moving
	[moveto X] - stop object X from moving
	[moveto X Y Z W*] - move object X to position Y,Z with speed W
	[move X Y Z*] - move object X in direction Y with speed Z
	[destroy] - destroy active object
	[destroy X] - destroy object X
	[create X Y*] - create object X with delay Y (or immediately if Y is ommited)
	[imageindex X Y] - set image_index of object X to Y (image_index is the current picture in the animation)
	[imagespeeds X Y] - set the default image speed for object X to Y
	[imagespeed X Y] - set current image speed for object X to Y
	[animations] - same as animation but continue after the animation ends
	[animation X] - set player animation to X
	[animation X Y] - animate to object X with sprite Y
	[animation X Y Z] - animate object X with sprite Y fixed on subimage Z
	[snoitamina] - same as animations but the animation is played in reverse
	[noitamina] - same as animation but the animation is played in reverse
	[type] - same as animation but plays a sound and animates the object using Y from map_pictures and map_sounds
	[silent] - same as type but without the sound playing
	[direction X] - set the player objects direction to X
	[talk X] - set the object X to the currently speaking object (which will set up its talking animation, block sound and speech bubble)
	[other] - same as [talk other]
	[player] - same as [talk player]
	[conversation X Y] - with the active object, start a dialogue X from section Y
	[end] - end the current conversation
	[section X] - jump to section X of the current conversation
	[mood X Y] - the object X is in mood Y (just for this game block) - the mood corresponds to different sprites and sounds when talking/listening
	[mood X] - the player is in mood X (just for this game block) - the player can be "angry", "scared"...
	[pickup X* Y*] - pick up the item from the active object from direction X ("left", "right", "back", "front") and height Y ("low", "middle", "high")
	[pick] - same as pickup but the object stays on the screen (it can be picked again)
	[take] - put item from currently active object into inventory
	[take X] - put item X into inventory
	[kick X* Y*] - the player kicks the object in direction X ("left", "right" or "auto" to figure it out automatically) with a breaking sound Y
	[draw SPRITE X Y TIME* DEPTH* SCALE* ALPHA* ANGLE* SPEED* DIR* ROT* SUBIMAGE* IMAGESPEED* DSCALE* DALPHA*] - draw a sprite on location X,Y
		notes:		
			time - for how long will the sprite appear on screen (or use the keyword "once" to show the sprite until it has finished its animation)
			depth, scale, alpha, angle - depth, image scale, image alpha and image angle of the sprite on screen
			speed, dir - the speed and direction at which the sprite is moving
			rot - the rotation at which the sprite is rotating
			subimage - the subimage the sprites animatino is fixed on
			imagespeed - the speed of the animation
			dscale - how much the image scale is changed each step (positive values mean the sprite gets larger)
			dalpha - how much the image alpha is changed each step (positive values mean the sprite gets more opaque)
	[room X Y] - go to room X (or keyword "previous" or "current") and use the entrance alias Y (or keyword "previous" or "current")
	[room X] - go to the last visited room with entrance X
	[room] - go to the last room with the last visited entrance
	[time X] - how many frames should the block last before going to the next block
	[time X Y] - the block will last as many frames as it takes for the player to reach object X with speed Y
	[time X Y Z] - the block will last as many frames as it takes for object X to reach object Y with speed Z
	[miliseconds] - same as time but with miliseconds instead of frames
	[stop] - stops execution of blocks until further notice from another object (for example until the minigame is done)
	[skip] - skip all game blocks
	[skip X] - move X game blocks ahead (or behind if X is negative)
	[set alias X] - set the player objects alias to X
	[set mood X] - set the default mood of the player to X
	[set mood X Y] - set the default mood of object X to Y
	[set randomboolean] - set the variable randomboolean randomly to 0 or 1
	[set randomreply X] - set the variable randomreply randomly to an integer from 0 to Y
	[set X] - set variable X to ""
	[set X Y] - set variable X to Y
	[add X Y] - add to variable X value Y
	[clear] - delete all variables from memory
	[clear X] - delete variable X from memory
	[if X] - parse the rest of this block if any of the conditions X are true
	[not X] - parse the rest of this block if any of the conditions X are not true
		notes:
			The if and not statements take conditions such as [if item umbrella] or [if coins = 5]
			They can also take multiple conditions and evaluate if any of them is true such as [if (item umbrella) or (coins = 5)]
			The conditions are:
			shaders - returns true if shaders are supported
			shader X - returns true if shader X is compiled
			occupied - returns true if the player is occupied
			occupied X - return true if the player objects occupied variable is X
			alias X - return true if the player objects alias variable is X
			item X - return true if item X is in the inventory
			mood X - return true if player objects mood variable is X
			random X - a chance to return true is 1/X
			chance X - same as random X
			version X - return true if the game version in setup.json is X
			available X - return true if object X is available
			X < Y - return true if variable X is smaller than Y
			X <= Y - return true if variable X is smaller or equal to Y
			X = Y - return true if variable X is equal to Y
			X == Y - same as X = Y
			(instead of X you can use the keyword "scale" which will check the player objects image scale variable)

---

conversations.txt - a text file consisting of all the conversations in the game that are activated using the [conversation] game block

These conversations vaguely resemble the rules of "ink", a program to write conversations.
Here is an illustrative example:

=== this_is_a_conversation_name ===
== this_is_a_section_name ==
	Every TAB means a change in roles.
	Even number of TABs means that the player is speaking.
		Odd number of TABs means that the other character is speaking.
			So this is the player speaking again.
			[talk someone_else] And this is a third person speaking...
			+ Dialogue options have a + in front
				Each dialogue is initiated only if the player selects that option
				After the dialogue is over, a block option should divert it to another section:
				[section this_is_a_section_name]
			+ If the dialogue option has one plus, the player will read it
				[section this_is_a_section_name]
			++ If the dialogue option has another plus, the player will not read it aloud.
					Instead, the player can read something else.
					[section this_is_a_section_name]
			- If the dialogue option has a minus, it will show up only once.
				[section this_is_a_section_name]
			-+ Same as before, an extra plus means the player will not read this line aloud.
				[section this_is_a_section_name]
			+[if item umbrella] Options can have conditions
				Options with conditions will only show to the player if the condition is met.
					+ There can be nested options
						[section this_is_a_section_name]
					+ And new sections can be identified anywhere
						== some_section ==
						Such as above.
						[section this_is_a_section_name]
					+ If there is only one option to choose from...
						Like in the example below:
							+ It will be selected automatically.
								[section this_is_a_section_name]
			+ Also, beware infinite loops.
				== this_is_a_loop ==
				Such as this one.
				[section this_is_a_loop]
			+ When the conversation is over, use the block option end:
				The game will continue where it left off before the conversation was initiated.
				[end]
always end the dialogue options tree with some text.

== some_completely_unrelated_section ==
			+ For example, if there was no line "always end the dialogue options tree with some text"...
				...then this unrelated line would also be included in the dialogue options
				Because it is indented the same number of times.
				[end]
end.
